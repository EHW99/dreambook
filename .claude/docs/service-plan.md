# 꿈꾸는 나 — 서비스 기획서

## 서비스 한 줄 요약

아이의 이름/생년월일/사진/직업을 입력하면, AI가 그 직업으로 활약하는 동화책을 만들어 실물 책으로 인쇄/배송하는 서비스.

---

## 핵심 정보

| 항목 | 내용 |
|------|------|
| 타겟 | 유아(5~7세) 부모, 유치원/어린이집 |
| 차별점 | 아이 사진 기반 캐릭터 + 직업 동화 개인화 |
| 판형 | SQUAREBOOK_HC (243×248mm, 하드커버) 기본, 선택 가능 |
| 페이지 | 최소 24p (선택 가능, 2p 단위 증가) |
| 책 구성 | 1권 = 1직업 = 1이야기 |

---

## 사용자 입력

| 입력 | 설명 |
|------|------|
| 아이 이름 | 동화 주인공 이름 |
| 생년월일 | "미래의 나" 스타일 시 연도 계산용, "꿈꾸는 오늘" 시 나이 표시용 |
| 사진 | AI가 아이를 닮은 캐릭터 일러스트 생성에 사용 |
| 직업 | 카테고리별 목록에서 선택 |
| 동화 스타일 | 꿈꾸는 오늘 / 미래의 나 |
| 페이지 수 | 24p~ (선택) |
| 줄거리 | 스토리 테마 선택 또는 직접 작성 |

## 동화 스타일 2종

### 꿈꾸는 오늘
- 아이 외형 그대로 유지 (체형, 얼굴) — 애들 만화처럼
- 직업 복장만 입힘
- "어느 날, 소방관이 된 민준이는..."
- 시간/나이 언급 없음, 꿈이 이루어진 세계
- **캐릭터 시트**: 1세트 (11장 전부 동일 캐릭터)
- **구현 난이도**: 낮음

### 미래의 나
- 성장 과정 반영 — 얼굴도 바뀌고 키도 큼
- 생년월일 기반 연도 계산 ("2036년, 25살이 된 민준이는...")
- 일대기 구조 (아이 → 초등학생 → 중학생 → 고등학생 → 성인)
- **캐릭터 시트**: 나이별 여러 세트 (원본 사진 기반 성장 모습 생성)
- **구현 난이도**: 높음 (나이별 일관성 유지 필요, API 호출 증가)

Phase 2에서는 두 옵션 모두 **UI만 구현** (더미 이미지). Phase 3에서 AI 연동 시 난이도에 따라 조정.

| 카테고리 | 직업 예시 |
|---------|---------|
| 🏥 사람을 도와요 | 의사, 소방관, 경찰관, 간호사 |
| 🎨 만들어요 | 화가, 요리사, 건축가, 패션디자이너 |
| 🔬 탐구해요 | 과학자, 우주비행사, 고고학자 |
| 🎤 보여줘요 | 가수, 운동선수, 유튜버 |
| 🌿 자연을 지켜요 | 수의사, 동물학자, 환경운동가 |
| 🤖 어떤 직업이 좋을지 모르겠어요 | 성향 테스트 → 추천 (Phase 3) |

## 스토리 테마 선택

직업 선택 후, 해당 직업의 스토리 테마를 제시한다.

예시 (경찰관):
- 🦸 정의의 수호자 — 범인을 쫓고 사건을 해결하는 형사 이야기
- 🏍️ 거리의 영웅 — 오토바이를 타고 교통을 지키는 교통경찰 이야기
- 🤝 마을의 친구 — 시민을 도우며 헌신하는 동네 경찰 이야기
- ✏️ 직접 쓸래요 — 원하는 줄거리를 직접 입력

각 줄거리 테마는 **테마 이름 + 간략한 줄거리**를 함께 보여준다.

**Phase 2 구현 범위**: "직접 쓸래요"만 동작. 나머지 줄거리 테마는 "준비 중" 배지 표시 + 클릭 시 "곧 제공될 예정입니다" 토스트 메시지.

## 책 구조 (24p 기준)

```
표지 (앞)
제목 페이지 (1장) — "소방관이 된 민준이"
내용 (10장) — 10개 장면
뒷표지 (뒤)
= 총 24p (양면 12장)
```

내용 11장의 구조:
- 제목 페이지: 책 제목 + 아이 캐릭터
- 1장: 도입 — 아이 소개
- 2~9장: 전개 — 직업 활동 장면
- 10장: 마무리 — 엔딩

페이지 수 늘리면 장면 수가 늘어남.

## 재생성 제한

| 대상 | 제한 | 설명 |
|------|------|------|
| 캐릭터 시트 | 최대 4회 | 주인공 캐릭터 미리보기 재생성 |
| 스토리 (전체 줄거리) | 최대 3회 | 전체 스크립트를 통으로 재생성 |
| 이미지 (페이지별) | 페이지당 최대 4회 | 해당 페이지 일러스트만 재생성 |

**갤러리 방식** (캐릭터, 이미지): 재생성 시 기존 결과물은 삭제되지 않음. 생성된 모든 결과물 중에서 선택하는 방식.
예: 캐릭터 2회 재생성 → 총 3개 중 마음에 드는 것 선택.

**스토리 재생성 시 연쇄 처리**: 스토리를 재생성하면 scene_description이 전부 바뀌므로:
- 기존 페이지별 이미지 전부 폐기
- 새 스토리 기반으로 이미지 자동 재생성
- 이미지 재생성 횟수 리셋 (페이지당 4회 다시 부여)
- 스토리는 갤러리 아님 — 교체 방식 (이전 스토리 유지 안 함)

UI에 남은 횟수 표시. 초과 시 "재생성 횟수를 모두 사용했습니다" 메시지.

## 사용자 플로우

```
0. 랜딩 페이지
   서비스 소개, 샘플 동화책 미리보기, 가격/이용권 안내
            ↓
1. 회원가입/로그인
            ↓
2. 이용권 구매 (목업)
   이용권 선택 → 결제 (목업, 실결제 없음)
   이용권이 있으면 이 단계 스킵
            ↓
3. 정보 입력
   아이 이름, 생년월일, 사진 업로드
   (기존 등록 사진에서 선택도 가능)
            ↓
4. 직업 선택
   카테고리별 목록에서 선택
            ↓
5. 동화 스타일 선택
   꿈꾸는 오늘 / 미래의 나
            ↓
6. 그림체 선택
   수채화 / 연필화 / 크레파스 / 3D / 만화 등
   각 스타일별 샘플 이미지 미리보기 제공
            ↓
7. 캐릭터 미리보기
   아이 사진 + 그림체 + 직업 복장 + 동화 스타일 반영
   → 캐릭터 시트 생성
   마음에 안 들면 재생성 (최대 4회, 갤러리 방식)
   확정해야 다음 단계로 진행
            ↓
8. 옵션 선택
   페이지 수, 판형
   예상 가격 실시간 표시
            ↓
9. 줄거리 작성
   스토리 테마 선택 (Phase 2: 직접 쓰기만 동작)
   또는 직접 줄거리 입력
            ↓
10. 생성 중
    AI 스토리 생성 → AI 이미지 생성
    원형 진행률 표시 + 예상 소요 시간 안내
            ↓
11. 편집/미리보기
    페이지별 일러스트 + 텍스트 확인
    줄거리(텍스트) 직접 수정 가능
    스토리 재생성 (3회) / 이미지 재생성 (페이지당 4회)
    전체 미리보기 (책 넘기기 뷰)
            ↓
12. 주문
    배송지 입력 (수령인, 주소, 전화번호)
    Book Print API로 책 생성 → 주문
```

## 디자인 방향

### 레퍼런스
- **참고 사이트**: https://storyclass.ai/ (AI스토리교실)
- 이 사이트의 전체적인 분위기를 참고하되, **반드시 차별화**할 것

### AI스토리교실 특징 (참고만)
- 보라색 계열 (#6133E2) 중심
- Noto Sans KR, 둥근 버튼 (border-radius 50px)
- 반응형, 마소니리 그리드 갤러리
- 깔끔하고 교육 친화적

### dreambook 디자인 — 차별화 포인트

**AI스토리교실과 절대 같으면 안 된다. 우리만의 특색을 반드시 살릴 것.**

| 항목 | AI스토리교실 | dreambook (우리) |
|------|------------|-----------------|
| 색상 | 보라색 계열, 차갑고 모던 | **따뜻한 파스텔 톤** (살몬 핑크, 복숭아, 아이보리) |
| 느낌 | 교육 플랫폼, 깔끔 | **가족의 온기, 부드럽고 포근한** |
| 전환 | 기본적 | **부드러운 페이지 전환 (Framer Motion)** |
| 일러스트 | 없음 | **따뜻한 손그림 일러스트 활용 (unDraw, 커스텀)** |
| 타이포 | Noto Sans KR | **둥근 느낌의 폰트 (Pretendard 또는 둥근 한글 폰트)** |

### 색상 팔레트

```
Primary:    #FFB5A7 (따뜻한 살몬 핑크)
Secondary:  #FCD5CE (연한 복숭아)
Accent:     #A8DADC (부드러운 민트)
Background: #FFF8F0 (따뜻한 아이보리)
Text:       #2D3436 (부드러운 차콜)
Success:    #B5EAD7 (연한 민트 그린)
Warning:    #FFE0AC (부드러운 옐로우)
```

### 디자인 원칙
- 모든 모서리 둥글게 (border-radius 12~20px)
- 그림자는 부드럽게 (box-shadow 투명도 낮게)
- 페이지 전환 부드럽게 (Framer Motion)
- 로딩/대기 시 따뜻한 애니메이션 (Lottie)
- 빈 상태 화면에 일러스트 활용
- 반응형 필수 (데스크톱/태블릿/모바일)

### UI 라이브러리
- shadcn/ui (기본 컴포넌트) + Tailwind CSS
- Framer Motion (페이지 전환, 카드 애니메이션)
- LottieFiles (로딩, 완성 축하 애니메이션)
- Lucide Icons (부드러운 라인 아이콘)

## 그림체 스타일

| 스타일 | 설명 | 프롬프트 키워드 |
|--------|------|---------------|
| 🎨 수채화 | 부드럽고 따뜻한 동화 느낌 | watercolor, soft tones |
| ✏️ 연필화 | 스케치풍 손그림 느낌 | pencil sketch, hand-drawn |
| 🖍️ 크레파스 | 아이가 직접 그린 듯한 느낌 | crayon drawing, childlike |
| 🧸 3D | 픽사/디즈니 스타일 입체감 | 3D render, Pixar style |
| 📺 만화 | 밝고 선명한 카툰 스타일 | cartoon, cel-shaded |

각 스타일마다 미리 생성해둔 **샘플 이미지**를 랜딩 페이지 및 선택 화면에서 보여준다.

## 캐릭터 미리보기

사진 업로드 + 그림체 선택 후, 본격 동화 생성 전에 **주인공 캐릭터를 먼저 확정**하는 단계.

- GPT Image의 images.edit으로 아이 사진 → 선택한 그림체의 캐릭터 시트 생성
- 사용자가 결과를 확인하고 승인 또는 재생성
- **재생성 최대 4회**
- 확정된 캐릭터 시트는 저장 → 이후 모든 페이지 생성 시 참조 이미지로 사용
- 이 단계에서 캐릭터가 확정되어야 다음 단계 진행 가능

## 이용권 (목업)

실서비스에서는 이용권 구매 후 동화책 제작이 가능한 구조.
과제에서는 **결제 UI만 구현하고 실결제는 없음**.

| 이용권 | 내용 | 가격 (목업) |
|--------|------|-----------|
| AI 스토리북 이용권 | AI 동화책 1권 편집 + PDF 제공 | 9,900원 |
| AI 스토리북 + 실물 책 | 위 + 실물 책 인쇄/배송 | 29,900원 |

- "새 동화책 만들기" 클릭 시 이용권 보유 여부 확인
- 이용권 없으면 → 이용권 선택 화면 → 결제 (목업: 버튼 누르면 바로 구매 완료)
- 이용권 있으면 → 바로 정보 입력 단계로

### 저장 정책

**단계 이동 시 (다음 버튼 클릭)**: 해당 단계까지의 데이터가 서버에 저장됨.
**단계 내부 작업 중**: 서버에 저장되지 않음. (예: 편집 화면에서 텍스트 수정 중)
**자동 저장 없음**: 사용자가 명시적으로 "다음" 또는 "저장" 버튼을 눌러야 저장됨.

- 이용권 사용 후 동화책 제작 시작 시점에 "작성중" 레코드 생성
- "다음" 버튼을 누를 때마다 해당 단계 데이터가 서버에 반영
- 브라우저 닫기 시: beforeunload 경고창 표시, 저장되지 않은 단계 내부 데이터는 유실
- 재접속 시: 마지막으로 "다음"을 눌렀던 단계부터 이어서 진행 가능

```
내 책장
├── 📕 소방관이 된 민준이 [완성]
├── 📙 요리사가 된 민준이 [작성중 - 캐릭터 선택까지 완료]
└── 📘 + 새 동화책 만들기
```

## 오디오북 (프로토타입)

완성된 동화책을 TTS로 읽어주는 기능.

- 일러스트를 화면에 띄우면서 TTS가 해당 페이지 텍스트를 읽어줌
- 자동 페이지 넘김 (읽기 완료 시)
- 재생/일시정지 컨트롤
- **Phase 2에서는 UI + 기본 TTS만 구현 (프로토타입)**
- 추후 감정 표현 있는 고급 TTS로 업그레이드 가능

## 공통 요구사항

### 접근 권한

| 페이지/기능 | 비로그인 | 로그인 |
|------------|---------|--------|
| 랜딩 페이지 | O | O |
| 샘플 동화책 보기 | O | O |
| 가격/이용권 안내 | O | O |
| 회원가입/로그인 | O | - |
| 새 동화책 만들기 | X → 로그인 페이지로 이동 | O |
| 마이페이지 | X → 로그인 페이지로 이동 | O |
| 내 책장 | X → 로그인 페이지로 이동 | O |

### 사진 업로드 검증
- 허용 형식: JPG, PNG, WebP
- 최대 파일 크기: 10MB
- 최소 해상도: 512x512 (AI 캐릭터 생성 품질 확보)
- 사용자당 최대 사진 수: 20장
- 저장: 서버 로컬 또는 클라우드 스토리지
- 삭제: 사진 삭제 시 서버에서 즉시 삭제 (이미 생성된 캐릭터 시트/일러스트는 유지)

### 반응형 디자인
- 데스크톱 + 태블릿 + 모바일 필수
- 부모+아이가 태블릿/스마트폰으로 함께 사용하는 상황 고려

### 이탈 방지
- 동화책 작성 중 브라우저 닫기/뒤로가기/탭 닫기 시 브라우저 기본 경고창 표시
- "사이트에서 나가시겠습니까? 변경사항이 저장되지 않을 수 있습니다." (나가기 / 취소)
- `beforeunload` 이벤트 활용
- 저장 정책은 상단 "저장 정책" 섹션 참고

### 뒤로가기 규칙

| 현재 단계 | 뒤로가기 가능 | 처리 |
|----------|-------------|------|
| 정보 입력 | O | 자유 이동 |
| 그림체 선택 | O | 자유 이동 |
| 캐릭터 미리보기 (확정 전) | O | 자유 이동 |
| 캐릭터 미리보기 (확정 후) → 직업 선택 이후 | O | 캐릭터 재선택 가능, 재생성 횟수 유지, 이후 단계 데이터 리셋 |
| 줄거리 작성 | O | 이전 단계 자유 이동, 줄거리 입력 내용 유지 |
| 생성 중 | X | 생성 진행 중에는 뒤로가기 차단 |
| 편집/미리보기 | O | 이전 단계로 돌아가면 생성된 콘텐츠 유지 |
| 주문 (배송지 입력) | O | 편집으로 돌아가기 가능, 편집 내용 유지, Book Print API 호출 전이므로 안전 |
| 주문 (결제 완료 후) | X | 주문 확정 후에는 돌아갈 수 없음, 취소만 가능 |

**핵심 원칙**: Book Print API 호출(책 생성/최종화)은 주문 확정 시점에 일괄 수행. 편집 단계까지는 서버 DB에만 저장되므로 자유롭게 이동 가능.

### 에러/빈 상태 페이지
- 404 페이지 (없는 페이지)
- 500 에러 페이지
- 빈 상태 UI ("아직 만든 동화책이 없어요", "등록된 사진이 없어요" 등)

### 주문 상태 알림
- 주문 후 상태 변경 추적 (Webhook 활용)
- 마이페이지 주문 내역에서 실시간 상태 표시

## Phase별 구현 범위

### Phase 2 (기반 서비스)
- 랜딩 페이지 (서비스 소개, 샘플, 가격 안내)
- 회원가입/로그인 (인증, JWT)
- 마이페이지 (회원 정보 수정, 탈퇴)
- 아이 사진 관리 (업로드, 목록, 삭제, 선택)
- 정보 입력 → 직업 선택 → 옵션 선택 → 줄거리 직접 작성
- 더미 스토리/이미지로 편집 화면 구현
- 책 뷰어 (페이지 플립, 전체화면)
- Book Print API 연동 (책 생성 → 사진 업로드 → 표지/내지 → 최종화 → 주문)
- 내 책장 (CRUD) + 주문 내역 조회
- 주문 상태 추적 (Webhook)
- 오디오북 프로토타입 (UI + 기본 TTS)
- 반응형 디자인 (데스크톱/태블릿/모바일)
- 이탈 방지 경고창
- 에러/빈 상태 페이지
- 스토리 테마 선택 UI (목업, 동작 안 함)
- 성향 테스트 UI (목업, 동작 안 함)

### Phase 3 (AI 기능)
- AI 스토리 생성 (GPT-4o)
- AI 이미지 생성 (GPT Image, 아이 사진 기반)
- 재생성 횟수 제한 적용
- 스토리 테마 선택 동작 연결
- 성향 테스트 → 직업 추천
