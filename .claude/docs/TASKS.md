## Phase 2: 기반 서비스

### 태스크 1: 프로젝트 초기 설정 + DB 스키마

- **설명**: Next.js(App Router) 프론트엔드와 FastAPI 백엔드의 프로젝트 구조를 생성하고, SQLite + SQLAlchemy 기반 DB 스키마를 구현한다. 환경변수 설정, CORS, 기본 미들웨어를 포함한다.
- **선행 태스크**: 없음
- **완료 기준**:
  1. `frontend/` — Next.js App Router 프로젝트 생성, Tailwind CSS + shadcn/ui 설정 완료
  2. `backend/` — FastAPI 프로젝트 생성, SQLAlchemy + SQLite 연결 완료
  3. SPEC.md의 8개 테이블 (users, photos, books, character_sheets, pages, page_images, vouchers, orders) 모델 정의 완료
  4. 마이그레이션 실행 시 테이블 생성 확인
  5. `.env.example` 파일에 필요한 환경변수 목록 포함 (BOOKPRINT_API_KEY, BOOKPRINT_BASE_URL, OPENAI_API_KEY, SECRET_KEY)
  6. CORS 설정 (프론트엔드 ↔ 백엔드 통신 허용)
  7. 프론트엔드 `npm run dev`, 백엔드 `uvicorn` 정상 기동 확인
  8. 색상 팔레트, 폰트 등 디자인 토큰을 Tailwind 설정에 반영

### 태스크 2: 회원가입 / 로그인 / 인증

- **설명**: 이메일+비밀번호 기반 회원가입, 로그인, JWT 인증 시스템을 구현한다. 프론트엔드 페이지와 백엔드 API를 모두 포함한다.
- **선행 태스크**: 태스크 1
- **완료 기준**:
  1. **백엔드 API**:
     - `POST /api/auth/signup` — 이메일 형식 검증, 이메일 중복 검증, 비밀번호 최소 8자 검증, bcrypt 해시 저장
     - `POST /api/auth/login` — 이메일+비밀번호 검증, Access Token(1시간) + Refresh Token(7일) 발급
     - `POST /api/auth/refresh` — Refresh Token으로 Access Token 갱신
     - `GET /api/auth/me` — 토큰으로 현재 사용자 정보 조회
  2. **에러 메시지**:
     - "이미 가입된 이메일입니다" (409)
     - "이메일 또는 비밀번호가 틀렸습니다" (401)
     - "비밀번호는 8자 이상이어야 합니다" (422)
     - "유효하지 않은 이메일 형식입니다" (422)
  3. **프론트엔드**:
     - 회원가입 페이지 (`/signup`) — 입력 폼, 유효성 검사, 에러 표시
     - 로그인 페이지 (`/login`) — 입력 폼, 에러 표시, 회원가입 링크
     - 로그인 성공 시 토큰 저장 + 리다이렉트
     - 인증이 필요한 페이지 접근 시 로그인 페이지로 리다이렉트
  4. 디자인: 파스텔 톤 색상 팔레트 적용, 둥근 모서리, 따뜻한 느낌
  5. 반응형: 모바일/태블릿/데스크톱

### 태스크 3: 마이페이지 — 회원 정보 + 탈퇴

- **설명**: 마이페이지의 회원 정보 탭을 구현한다. 비밀번호 변경, 회원 탈퇴 기능을 포함한다.
- **선행 태스크**: 태스크 2
- **완료 기준**:
  1. **백엔드 API**:
     - `PATCH /api/users/password` — 현재 비밀번호 확인 + 새 비밀번호 검증(8자 이상) + 변경
     - `DELETE /api/users/me` — 회원 탈퇴 (CASCADE: 사용자의 모든 데이터 즉시 삭제 — 사진 파일, DB 레코드)
  2. **에러 메시지**:
     - "현재 비밀번호가 틀렸습니다" (401)
     - "새 비밀번호는 8자 이상이어야 합니다" (422)
  3. **프론트엔드**:
     - 마이페이지 레이아웃 (탭 UI: 회원 정보 / 아이 사진 / 내 책장 / 주문 내역)
     - 회원 정보 탭: 이메일(비활성), 비밀번호 변경 폼
     - 회원 탈퇴: 확인 다이얼로그 ("정말 탈퇴하시겠습니까? 모든 데이터가 삭제됩니다")
     - 탈퇴 성공 → 토큰 삭제 → 랜딩 페이지 이동
  4. 디자인: 마이페이지 일관된 레이아웃, 반응형

### 태스크 4: 아이 사진 관리

- **설명**: 마이페이지의 아이 사진 관리 탭을 구현한다. 사진 업로드, 목록 조회, 삭제를 포함한다.
- **선행 태스크**: 태스크 3
- **완료 기준**:
  1. **백엔드 API**:
     - `POST /api/photos` — 파일 업로드 (multipart/form-data)
       - 형식 검증: JPG, PNG, WebP만 허용
       - 크기 검증: 10MB 이하
       - 해상도 검증: 512x512 이상
       - 개수 검증: 사용자당 최대 20장
       - 서버 로컬 저장 (uploads/ 디렉토리)
     - `GET /api/photos` — 내 사진 목록 (id, 썸네일 URL, 원본 파일명, 업로드일)
     - `DELETE /api/photos/:id` — 사진 삭제 (DB + 파일 즉시 삭제)
       - 본인 사진만 삭제 가능 (403)
  2. **에러 메시지**:
     - "지원하지 않는 파일 형식입니다" (422)
     - "파일 크기가 10MB를 초과합니다" (422)
     - "최대 20장까지 등록 가능합니다" (422)
     - "최소 512x512 이상의 이미지를 업로드해주세요" (422)
  3. **프론트엔드**:
     - 사진 목록 그리드 (썸네일 + 파일명 + 날짜)
     - 업로드 버튼 + 드래그앤드롭 지원
     - 삭제 버튼 + 확인 다이얼로그
     - 빈 상태: "등록된 사진이 없어요" + 업로드 안내
  4. 정적 파일 서빙 (업로드된 사진 접근용)
  5. 반응형

### 태스크 5: 랜딩 페이지

- **설명**: 서비스의 첫 인상을 결정하는 랜딩 페이지를 구현한다. 서비스 소개, 그림체 샘플, 가격 안내를 포함한다.
- **선행 태스크**: 태스크 1
- **완료 기준**:
  1. **히어로 섹션**: 서비스 핵심 메시지 + CTA 버튼 ("동화책 만들기")
  2. **샘플 동화책 섹션**: 미리 준비된 샘플 동화 카드 (클릭 시 책 뷰어로 이동, Phase 2에서는 더미)
  3. **그림체 샘플 섹션**: 5가지 그림체(수채화/연필화/크레파스/3D/만화) 샘플 이미지 갤러리
  4. **가격/이용권 섹션**: 이용권 2종 카드 (AI 스토리북 9,900원 / +실물 책 29,900원)
  5. **접근 권한**: 비로그인/로그인 모두 접근 가능
  6. **CTA 버튼**: 비로그인 → 로그인 페이지, 로그인 → 동화책 만들기 페이지
  7. **헤더/네비게이션**: 로고, 로그인/회원가입 또는 마이페이지 버튼
  8. **푸터**: 서비스명, 회사 정보
  9. **디자인**: 파스텔 톤, Framer Motion 애니메이션, 스크롤 기반 페이드인
  10. **반응형**: 모바일/태블릿/데스크톱 완벽 대응

### 태스크 6: 동화책 만들기 위자드 — 정보 입력 + 직업 선택

- **설명**: 동화책 만들기 위자드의 첫 2단계(정보 입력, 직업 선택)를 구현한다.
- **선행 태스크**: 태스크 2, 태스크 4
- **완료 기준**:
  1. **이용권 확인**: "새 동화책 만들기" 클릭 시 이용권 보유 여부 확인 → 없으면 이용권 구매 UI로 이동
  2. **이용권 구매 UI (목업)**: 이용권 2종 선택 → 구매 버튼 클릭 → 즉시 구매 완료
  3. **위자드 레이아웃**: 진행 표시 바 (전체 단계 시각화), 뒤로가기/다음 버튼
  4. **정보 입력 단계**:
     - 아이 이름 입력 (필수, 최대 20자)
     - 생년월일 선택 (필수)
     - 사진 선택: 기존 등록 사진에서 선택 또는 새로 업로드
     - 입력 검증 + 에러 표시
  5. **직업 선택 단계**:
     - 카테고리별 직업 목록 (5개 카테고리 + "어떤 직업이 좋을지 모르겠어요")
     - 카테고리 탭/아코디언 UI
     - 직업 카드 선택 UI (아이콘 + 직업명)
     - "어떤 직업이 좋을지 모르겠어요" → "준비 중" 배지 표시
  6. **백엔드**:
     - `POST /api/books` — 동화책 레코드 생성 (status: draft, 이용권 연결)
     - `PATCH /api/books/:id` — 정보 입력/직업 선택 저장
     - `POST /api/vouchers/purchase` — 이용권 구매 (목업)
     - `GET /api/vouchers` — 이용권 목록 조회
  7. **저장 정책**: "다음" 버튼 시 서버 저장
  8. **beforeunload** 이벤트 등록 (이탈 방지)
  9. 디자인, 반응형

### 태스크 7: 동화책 만들기 위자드 — 스타일 선택 + 캐릭터 미리보기

- **설명**: 위자드의 동화 스타일, 그림체 선택, 캐릭터 미리보기 단계를 구현한다.
- **선행 태스크**: 태스크 6
- **완료 기준**:
  1. **동화 스타일 선택**:
     - "꿈꾸는 오늘" / "미래의 나" 카드 선택 UI
     - 각 스타일 설명 텍스트 표시
  2. **그림체 선택**:
     - 5가지 스타일 카드 (수채화/연필화/크레파스/3D/만화)
     - 각 스타일별 더미 샘플 이미지 미리보기
  3. **캐릭터 미리보기**:
     - "캐릭터 생성하기" 버튼 → Phase 2에서는 더미 캐릭터 이미지 반환
     - 생성된 캐릭터 시트 표시
     - 재생성 버튼 + 남은 횟수 표시 (최대 4회)
     - 갤러리 방식: 생성된 모든 캐릭터 시트 나열 → 하나 선택
     - "확정하기" 버튼 → 캐릭터 확정 후 다음 단계 진행 가능
  4. **백엔드**:
     - `PATCH /api/books/:id` — 스타일/그림체 저장
     - `POST /api/books/:id/character` — 캐릭터 시트 생성 (더미 이미지 반환)
     - `GET /api/books/:id/characters` — 캐릭터 시트 갤러리 조회
     - `PATCH /api/books/:id/character/:charId/select` — 캐릭터 선택
     - 재생성 횟수 초과 시 에러: "재생성 횟수를 모두 사용했습니다" (422)
  5. **뒤로가기**: 캐릭터 확정 후 이전 단계 돌아갈 시 캐릭터 재선택 가능, 재생성 횟수 유지
  6. 디자인, 반응형

### 태스크 8: 동화책 만들기 위자드 — 옵션 + 줄거리 + 생성

- **설명**: 위자드의 옵션 선택, 줄거리 작성, 생성 단계를 구현한다.
- **선행 태스크**: 태스크 7
- **완료 기준**:
  1. **옵션 선택**:
     - 페이지 수 선택 (24p~, 2p 단위 증가, 슬라이더 또는 셀렉트)
     - 판형 선택 (SQUAREBOOK_HC 기본, PHOTOBOOK_A4_SC, PHOTOBOOK_A5_SC)
     - 예상 가격 실시간 표시 (가격 계산 공식: priceBase + 추가 페이지 비용)
  2. **줄거리 작성**:
     - 스토리 테마 선택 UI (직업별 테마 카드 3~4개 + "직접 쓸래요")
     - Phase 2: "직접 쓸래요"만 활성화, 나머지는 "준비 중" 배지 + 클릭 시 토스트 "곧 제공될 예정입니다"
     - "직접 쓸래요" 선택 시 텍스트 입력 필드 (textarea)
  3. **생성 중**:
     - Phase 2: 더미 스토리 + 더미 이미지를 사용하여 즉시 페이지 데이터 생성
     - 원형 진행률 UI (Lottie 애니메이션)
     - "생성 중" 상태에서 뒤로가기 차단
     - 생성 완료 → 자동으로 편집 화면 이동
  4. **백엔드**:
     - `PATCH /api/books/:id` — 옵션/줄거리 저장
     - `POST /api/books/:id/generate` — 더미 스토리+이미지 생성
       - 더미 스토리: 템플릿 기반 (아이 이름, 직업 치환)
       - 더미 이미지: 미리 준비된 placeholder 이미지
       - pages 테이블에 페이지 데이터 삽입
       - page_images 테이블에 이미지 데이터 삽입
  5. 디자인, 반응형

### 태스크 9: 편집 / 미리보기 + 책 뷰어

- **설명**: 생성된 동화책을 편집하고, 전체 미리보기(책 넘기기 뷰)로 확인하는 기능을 구현한다.
- **선행 태스크**: 태스크 8
- **완료 기준**:
  1. **편집 화면**:
     - 페이지 목록 (썸네일 사이드바 또는 하단 네비게이션)
     - 선택된 페이지: 일러스트 + 텍스트 표시
     - 텍스트 인라인 편집 (클릭 → 수정 → 저장)
     - 스토리 재생성 버튼 (남은 횟수 표시, 최대 3회)
       - Phase 2: 재생성 시 동일 더미 반환, 횟수만 카운트
       - 재생성 시 연쇄 처리: 기존 이미지 폐기 + 새 이미지 생성 + 이미지 재생성 횟수 리셋
     - 페이지별 이미지 재생성 버튼 (남은 횟수 표시, 최대 4회, 갤러리 방식)
     - "재생성 횟수를 모두 사용했습니다" 비활성화 + 메시지
  2. **전체 미리보기 (책 뷰어)**:
     - 페이지 플립 애니메이션 (책 넘기기 효과)
     - 좌우 페이지 표시
     - 전체화면 모드
     - 페이지 네비게이션 (이전/다음)
  3. **백엔드**:
     - `GET /api/books/:id/pages` — 전체 페이지 조회 (텍스트 + 선택된 이미지)
     - `PATCH /api/books/:id/pages/:pageId` — 텍스트 수정
     - `POST /api/books/:id/regenerate-story` — 스토리 재생성 (더미, 횟수 체크)
     - `POST /api/books/:id/pages/:pageId/regenerate-image` — 이미지 재생성 (더미, 횟수 체크)
     - `GET /api/books/:id/pages/:pageId/images` — 이미지 갤러리
     - `PATCH /api/books/:id/pages/:pageId/images/:imgId/select` — 이미지 선택
  4. 디자인, 반응형

### 태스크 10: Book Print API 연동 + 주문

- **설명**: Book Print API와 연동하여 책 생성 → 사진 업로드 → 표지/내지 → 최종화 → 견적 → 주문까지의 전체 워크플로우를 구현한다.
- **선행 태스크**: 태스크 9
- **완료 기준**:
  1. **백엔드 — Book Print API 연동 서비스**:
     - Sandbox 충전금 확인 + 부족 시 자동 충전 (`POST /credits/sandbox/charge`)
     - 책 생성 (`POST /books`) — bookSpecUid 하드코딩
     - 사진 업로드 (`POST /books/{bookUid}/photos`) — 일러스트 이미지들
     - 템플릿 조회 (`GET /templates?bookSpecUid=...`) — 사용 가능 템플릿 확보
     - 표지 생성 (`POST /books/{bookUid}/cover`)
     - 내지 삽입 반복 (`POST /books/{bookUid}/contents?breakBefore=page`)
     - 책 최종화 (`POST /books/{bookUid}/finalization`)
     - 견적 조회 (`POST /orders/estimate`)
     - 주문 생성 (`POST /orders`)
  2. **주문 페이지 (프론트엔드)**:
     - 배송지 입력 폼 (수령인, 전화번호, 우편번호, 주소, 상세주소, 배송 메모)
     - 입력 검증 (필수 필드, 전화번호 형식, 우편번호)
     - 견적 표시 (총 금액, 배송비, 포장비)
     - "주문하기" 버튼 → 전체 Book Print API 워크플로우 실행
     - 주문 진행 중 로딩 표시
     - 주문 완료 화면 (주문 번호, 축하 애니메이션)
  3. **에러 처리**:
     - 충전금 부족: 자동 충전 → 재시도
     - API 오류: 사용자에게 안내 ("주문 처리 중 오류가 발생했습니다")
     - 타임아웃: 재시도 안내
  4. **백엔드 API**:
     - `POST /api/books/:id/estimate` — 견적 조회
     - `POST /api/books/:id/order` — 주문 생성 (전체 워크플로우)
  5. Sandbox PAID 상태 정지 인지 (이후 상태 전이 없음)

### 태스크 11: 내 책장 + 주문 내역 + 주문 관리

- **설명**: 마이페이지의 내 책장, 주문 내역 탭을 구현한다. 주문 취소, 배송지 변경을 포함한다.
- **선행 태스크**: 태스크 10
- **완료 기준**:
  1. **내 책장 (프론트엔드)**:
     - 동화책 목록 카드 (썸네일, 제목, 상태 배지: 작성중/완성/주문됨)
     - 작성중인 책: [이어서 만들기] 버튼 → 마지막 저장 단계로 이동
     - 완성된 책: [보기] → 책 뷰어, [주문하기] → 주문 페이지
     - 작성중인 책 삭제 (완성/주문된 책은 삭제 불가)
     - "+ 새 동화책 만들기" 카드
     - 빈 상태: "아직 만든 동화책이 없어요"
  2. **주문 내역 (프론트엔드)**:
     - 주문 목록 (날짜, 책 제목, 상태, 가격)
     - 주문 상세: 배송 정보, 주문 상태, 운송장 번호
     - 주문 취소 버튼 (PAID/PDF_READY 상태만 표시)
     - 배송지 변경 버튼 (PAID~CONFIRMED 상태만 표시)
  3. **백엔드 API**:
     - `GET /api/books` — 내 동화책 목록
     - `GET /api/books/:id` — 동화책 상세
     - `DELETE /api/books/:id` — 작성중 동화책 삭제
     - `GET /api/orders` — 내 주문 목록
     - `GET /api/orders/:id` — 주문 상세
     - `POST /api/orders/:id/cancel` — 주문 취소 (Book Print API 연동)
     - `PATCH /api/orders/:id/shipping` — 배송지 변경 (Book Print API 연동)
  4. 디자인, 반응형

### 태스크 12: 웹훅 + 주문 상태 추적

  3. **테스트**:
     - `POST /webhooks/test` 로 테스트 이벤트 전송 가능
  4. Sandbox 환경에서 PAID 이후 상태 전이 없음을 인지하고 처리

## Phase 3: AI 기능 통합

### 태스크 15: AI 스토리 생성 (GPT-4o)

- **설명**: 더미 스토리 생성을 GPT-4o API 호출로 교체한다.
- **선행 태스크**: 태스크 8 (Phase 2 완료 후)
- **완료 기준**:
  1. `POST /api/books/:id/generate`의 스토리 생성 로직을 GPT-4o 호출로 교체
  2. **입력**: 아이 이름, 직업, 동화 스타일(꿈꾸는 오늘/미래의 나), 줄거리, 페이지 수
  3. **출력**: 페이지별 text_content + scene_description (이미지 생성 프롬프트)
  4. **프롬프트**: ai-guide.md 규칙에 따라 설계
  5. 스토리 재생성 (`POST /api/books/:id/regenerate-story`) GPT-4o 호출로 교체
  6. 재생성 횟수 체크 (최대 3회)
  7. 에러 처리: API 실패 시 사용자 안내, 타임아웃 처리

---

### 태스크 16: AI 캐릭터 시트 생성 (GPT Image)

- **설명**: 더미 캐릭터 시트를 GPT Image API images.edit 호출로 교체한다.
- **선행 태스크**: 태스크 7 (Phase 2 완료 후)
- **완료 기준**:
  1. `POST /api/books/:id/character`를 GPT Image images.edit으로 교체
  2. **입력**: 아이 사진 (참조 이미지) + 그림체 + 직업 + 동화 스타일
  3. **출력**: 1024x1024 PNG 캐릭터 시트
  4. **프롬프트**: 그림체별 키워드 (수채화→"watercolor, soft tones" 등), 직업 복장 묘사
  5. 생성된 이미지 서버 저장 + character_sheets 테이블 저장
  6. 재생성 횟수 체크 (최대 4회, 갤러리 방식)
  7. 에러 처리: 콘텐츠 필터링 거부, 타임아웃 등

### 태스크 17: AI 페이지 일러스트 생성 (GPT Image)

- **설명**: 더미 일러스트를 GPT Image API images.edit 호출로 교체한다.
- **선행 태스크**: 태스크 15, 태스크 16
- **완료 기준**:
  1. `POST /api/books/:id/generate`의 이미지 생성 로직을 GPT Image images.edit으로 교체
  2. **입력**: 캐릭터 시트 (참조 이미지) + scene_description + 그림체 키워드
  3. **출력**: 1024x1024 PNG 일러스트
  4. 페이지별 일러스트 순차 생성 (전체 페이지 수만큼)
  5. 이미지 재생성 (`POST /api/books/:id/pages/:pageId/regenerate-image`) 실제 AI 호출
  6. 재생성 횟수 체크 (페이지당 최대 4회, 갤러리 방식)
  7. **스토리 재생성 연쇄 처리**: 스토리 재생성 시 기존 이미지 폐기 → 새 일러스트 자동 생성 → 이미지 재생성 횟수 리셋
  8. 생성 진행률 표시 (프론트엔드 진행률 바와 연동)
  9. 에러 처리

### 태스크 18: AI 생성 통합 테스트 + 성향 테스트 UI

- **설명**: AI 스토리 + 캐릭터 + 일러스트의 전체 흐름을 통합 테스트하고, 성향 테스트/스토리 테마 선택 UI를 연결한다.
- **선행 태스크**: 태스크 17
- **완료 기준**:
  1. **E2E 테스트**: 정보 입력 → 직업 → 스타일 → 그림체 → 캐릭터 생성 → 확정 → 줄거리 → AI 생성 → 편집 → 주문 전체 흐름 동작
  2. **스토리 테마 선택 연결**: 테마 카드 클릭 시 해당 테마를 줄거리로 전달 → AI 스토리 생성에 반영
  3. **성향 테스트 UI**: "어떤 직업이 좋을지 모르겠어요" → 간단한 성향 질문 (3~5개) → 직업 추천 (AI 또는 규칙 기반)
  4. 재생성 전체 흐름 테스트 (캐릭터 4회, 스토리 3회, 이미지 페이지당 4회)
  5. 비용 모니터링 로그 (API 호출 횟수, 토큰 사용량)
