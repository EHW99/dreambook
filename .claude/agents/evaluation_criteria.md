# 평가 기준표

Developer와 Reviewer가 공유하는 품질 기준.
Developer는 이 기준을 미리 숙지하고 구현하고, Reviewer는 이 기준으로 채점한다.

---

## 채점 항목

### 1. 기능 완성도 (비중: 30%)

SPEC에 정의된 기능이 빠짐없이 동작하는가?

- **8~10점**: 모든 기능 구현 완료, 엣지 케이스 처리됨
- **5~7점**: 핵심 기능은 되지만 일부 누락 또는 불완전
- **3~4점**: 주요 기능 미구현 또는 동작 안 함
- **1~2점**: 대부분 기능 미구현

### 2. 코드 품질 (비중: 25%)

보안, 에러 처리, 코드 구조가 적절한가?

- **8~10점**: 보안 이슈 없음, 에러 처리 완비, 구조 명확
- **5~7점**: 기본 에러 처리 있으나 일부 허점
- **3~4점**: 보안 취약점 존재, 에러 시 500 발생
- **1~2점**: 하드코딩, 보안 무시, 구조 엉망

확인 포인트:
- API Key가 코드에 하드코딩되어 있지 않은가
- SQL Injection, XSS 등 보안 취약점이 없는가
- 입력 유효성 검사가 되어 있는가
- 에러 시 적절한 HTTP 상태 코드를 반환하는가
- 코드 중복이 과도하지 않은가

### 3. API 연동 (비중: 20%)

Book Print API와 Orders API가 정확히 연동되는가?

- **8~10점**: 전체 워크플로우 정상 동작 (책 생성 → 업로드 → 최종화 → 주문)
- **5~7점**: 일부 단계 동작하나 전체 흐름 미완
- **3~4점**: API 호출은 하나 에러 발생
- **1~2점**: 연동 시도 없음 또는 완전 실패

확인 포인트:
- 인증 헤더가 올바르게 설정되는가
- API 응답 에러를 적절히 처리하는가
- Sandbox 환경 제약사항을 고려했는가 (PAID 상태 정지 등)
- 판형 UID를 올바르게 사용하는가

### 4. 디자인 품질 (비중: 25%)

색상, 타이포그래피, 레이아웃, 애니메이션, 반응형이 일관되고 완성도 높은가?

- **8~10점**: 일관된 색상 팔레트, 부드러운 전환 효과, 반응형 완벽, 따뜻한 가족 서비스 느낌
- **5~7점**: 기본 흐름은 되나 디자인 일관성 부족, 애니메이션 없음
- **3~4점**: 색상 산만, 레이아웃 깨짐, Bootstrap 기본 느낌
- **1~2점**: 스타일링 거의 없음, 사용 불가 수준

확인 포인트:
- 파스텔/따뜻한 톤의 일관된 색상 팔레트를 사용하는가
- 타이포그래피가 통일되어 있는가 (2~3종 이내)
- 페이지 전환/카드 애니메이션이 부드러운가
- 모바일/태블릿/데스크톱 반응형이 정상 동작하는가
- 로딩/에러/빈 상태 UI가 디자인과 일관되는가
- 아동/가족 서비스에 어울리는 친근한 느낌인가

---

## 판정 기준

가중 점수 = (기능 × 0.3) + (코드 × 0.25) + (API × 0.2) + (디자인 × 0.25)

| 점수 | 판정 | 조치 |
|------|------|------|
| 7.0 이상 | **PASS** | 다음 태스크로 진행 |
| 5.0 ~ 6.9 | **CONDITIONAL** | 피드백 반영 후 재검수 |
| 5.0 미만 | **FAIL** | 피드백 반영 후 재검수 |
| 기능 완성도 4점 이하 | **AUTO-FAIL** | 점수 무관 자동 불합격 |
