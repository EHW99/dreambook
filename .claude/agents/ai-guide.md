## 전체 AI 파이프라인

```
1. 캐릭터 시트 생성 (1회)
   아이 사진 + 선택한 그림체 → images.edit → 캐릭터 시트
   사용자 확인 → 재생성 가능 (최대 4회, 갤러리 방식)
   확정된 캐릭터 시트 저장
        ↓
2. 스토리 생성
   아이 이름 + 생년월일 + 직업 + 동화 스타일 + 줄거리
   → GPT-4o → 페이지별 텍스트 (JSON 배열)
        ↓
3. 페이지별 일러스트 생성
   확정된 캐릭터 시트 (참조 이미지) + 장면 묘사
   → images.edit → 각 페이지 일러스트
        ↓
4. 사용자 편집
   텍스트 직접 수정 가능
   이미지 재생성 (페이지당 최대 4회, 갤러리 방식)
   스토리 전체 재생성 (최대 3회)
```

## 그림체 스타일

사용자가 선택하는 일러스트 스타일. 프롬프트에 반영됨.

| 스타일 | 프롬프트 키워드 |
|--------|---------------|
| 🎨 수채화 | watercolor illustration, soft warm tones, gentle brushstrokes |
| ✏️ 연필화 | pencil sketch, hand-drawn, fine line art |
| 🖍️ 크레파스 | crayon drawing, childlike texture, bold colors |
| 🧸 3D | 3D render, Pixar style, soft lighting, rounded shapes |
| 📺 만화 | cartoon style, cel-shaded, vibrant colors, clean outlines |

## 캐릭터 시트 생성

### 목적
아이 사진을 선택한 그림체로 변환하여, 이후 모든 페이지에서 참조 이미지로 사용.
캐릭터 일관성의 핵심.

### API 호출
```
엔드포인트: POST /v1/images/edits
image: 아이 원본 사진
prompt: "이 아이를 [선택한 그림체] 스타일의 동화 일러스트 캐릭터로 변환해줘.
         정면 포즈. 전신. 배경은 단색.
         얼굴 특징을 최대한 유지하되 일러스트 스타일로."
size: 1024x1024
quality: medium
output_format: png
```

### 동화 스타일별 캐릭터 시트 차이

**꿈꾸는 오늘**: 캐릭터 시트 1세트. 아이 모습 그대로 직업 복장만 변경.
**미래의 나**: 나이별 캐릭터 시트 여러 세트 필요 (아이→청소년→성인). 원본 사진 기반으로 성장 모습 생성.

### 갤러리 방식
- 재생성 시 기존 결과 삭제하지 않음
- 생성된 모든 결과물 중에서 선택
- 최대 4회 재생성 → 최대 5개 중 선택

## 스토리 생성 (GPT-4o)

### 필수 규칙
- 타겟 연령: 5~7세. 해당 연령이 이해할 수 있는 어휘만 사용
- 아이의 이름과 특징이 주인공에 자연스럽게 반영되어야 함
- 선택한 직업의 실제 활동을 구체적으로 묘사 (교육적 가치)
- 페이지 수에 맞는 기승전결 구조
- 각 페이지는 하나의 장면 — 일러스트로 그릴 수 있는 구체적 상황

### 동화 스타일별 스토리 구조

**꿈꾸는 오늘**:
```
"어느 날, 소방관이 된 민준이는..."
- 시간/나이 언급 없음
- 꿈이 이루어진 세계 설정
- 아이 모습 그대로 직업 활동
```

**미래의 나**:
```
"2036년, 25살이 된 민준이는..."
- 생년월일 기반 연도 계산
- 성장 과정 단계별 묘사
- 일대기 구조
```

### 금지 사항
- 폭력, 공포, 부정적 결말
- 성인 수준의 어휘나 개념
- 비현실적인 직업 묘사 (소방관이 불을 마법으로 끄는 등)
- 성별/인종 고정관념

### 프롬프트 구조
```
역할: 5~7세 아동용 동화 작가
입력: 아이 이름, 생년월일, 선택한 직업, 동화 스타일, 사용자 줄거리
출력: 페이지별 텍스트 (JSON 배열)
      [{"page": 1, "text": "...", "scene_description": "..."}, ...]
제약: [위 필수 규칙 + 금지 사항]
```

scene_description은 이미지 생성 프롬프트에 사용됨.

## 페이지별 일러스트 생성 (GPT Image)

### API 호출
```
엔드포인트: POST /v1/images/edits
image: 확정된 캐릭터 시트 (참조 이미지)
prompt: "[선택한 그림체] 스타일.
         장면: [scene_description]
         캐릭터: [아이 특징 — 캐릭터 시트 참조]
         구도: 텍스트 배치 영역을 하단/상단에 확보.
         배경: 장면에 맞는 구체적 배경."
size: 1024x1024
quality: medium
output_format: png
```

### 일관성 유지 규칙
- 매 페이지 프롬프트에 **그림체 키워드 동일하게** 포함
- 캐릭터 시트를 **매번 참조 이미지로** 전달
- 캐릭터 외형 묘사를 프롬프트에 **매번 포함** ("갈색 단발머리, 빨간 소방관 모자")

### Anti-Slop 금지 목록
- 과도하게 매끈한 3D 렌더링 느낌 (3D 스타일 선택 시 제외)
- 눈이 비정상적으로 큰 애니메이션 스타일
- 배경 없는 캐릭터만 떠있는 이미지
- 페이지마다 완전히 다른 그림체
- 성인 체형 (꿈꾸는 오늘 스타일 시)

### 갤러리 방식
- 페이지별 재생성 시 기존 결과 유지
- 생성된 것들 중 선택
- 페이지당 최대 4회 재생성

---

## 재생성 제한 요약

| 대상 | 제한 | 갤러리 |
|------|------|--------|
| 캐릭터 시트 | 최대 4회 | O (최대 5개 중 선택) |
| 스토리 전체 | 최대 3회 | X (교체) |
| 페이지별 이미지 | 페이지당 최대 4회 | O (최대 5개 중 선택) |

## 비용 절약 전략

1. **프롬프트를 구체적으로** — 모호하면 재생성 필요 → 비용 2배
2. **스토리 먼저, 이미지 나중** — 스토리 확정 후 이미지 생성 (순서 바뀌면 불일치)
3. **테스트 시 mock 사용** — 개발/테스트 단계에서는 더미 텍스트+샘플 이미지 사용
4. **실제 API 호출은 최종 확인 시만** — 불필요한 호출 최소화
5. **quality: medium 사용** — high 대비 1/4 가격, 동화 일러스트에 충분
6. **캐릭터 시트 재활용** — 같은 아이의 다른 책 만들 때 기존 캐릭터 시트 재사용 가능
